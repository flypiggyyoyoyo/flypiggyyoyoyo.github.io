数据的增删改查

在 MySQL 中，增删改查（即 CRUD 操作）是最基本的数据库操作，分别对应 插入（Insert）、删除（Delete）、更新（Update） 和 查询（Select） 操作。

1. 插入数据（INSERT）

INSERT 用于向表中插入新数据。
当你插入数据时，可以选择插入所有列的值，也可以只插入部分列的值。这个过程中，列的顺序和对应的 VALUES 语句中的值必须一致。若某些列没有提供值，可以使用 默认值 或 NULL（如果该列允许 NULL）。
只有在插入的值与表中的所有列顺序完全匹配，且插入的值覆盖表中的所有列，才可以省略列名，否则会出现错误。

基本语法：

  • table_name：要插入数据的表名。
  • column1, column2, column3, ...：列名，指定要插入数据的列
  • value1, value2, value3, ...：要插入的具体值，按列的顺序提供。

约束

MySQL 中的 约束（Constraints）是用于定义数据表中数据的规则，确保数据的有效性和一致性。常见的约束类型包括 主键约束、外键约束、唯一约束、检查约束、默认约束 等。

1. 主键约束（PRIMARY KEY）

  • 主键用于唯一标识表中的每一行。
  • 一个表只能有一个主键。
  • 主键列的值必须唯一，并且不能为 NULL。

示例：

在这个例子中，id 列是主键，保证每个 id 的值唯一且不能为空。

2. 外键约束（FOREIGN KEY）

  • 外键用于确保表与表之间的关系。
  • 外键列的值必须是另一个表中主键列的值（或者是有唯一约束的列）。
  • 外键约束确保引用的完整性，即不会在子表中插入无效的引用值。

示例：

在这个例子中，orders 表中的 customer_id 列是一个外键，引用了 customers 表中的 customer_id 列。

3. 唯一约束（UNIQUE）

  • 唯一约束确保列中的每个值都是唯一的。
  • 与主键不同，列可以包含 NULL 值，但不能包含重复的非 NULL 值。

示例：

4. 检查约束（CHECK）

  • 检查约束用于限制列的值。只有满足指定条件的数据才能被插入或更新。
  • 在 MySQL 8.0 及以上版本中支持 CHECK 约束。

5. 默认约束（DEFAULT）

  • 默认约束用于指定列的默认值，当插入数据时如果没有指定该列的值，则使用默认值。

6. 非空约束（NOT NULL）

  • 非空约束用于确保列中的值不能为空。也就是说，在插入或更新数据时，该列必须有值。

  • 用于数字列（通常是主键列），每次插入一行新记录时，自动生成唯一的递增值。
  • AUTO_INCREMENT 通常与主键一起使用。

正则表达式

在数据库中，正则表达式 通常用于匹配和查询符合特定模式的字符串数据。在 MySQL 数据库中，正则表达式通过 REGEXP 或 RLIKE 关键字来实现。

语法

  • REGEXP：用于匹配正则表达式。
  • pattern：正则表达式模式，用于匹配字符串。

RLIKE 是 REGEXP 的别名，两者功能完全相同。

正则表达式的基本规则

符号含义示例
.匹配任意一个字符a.b：匹配 “acb”、”a1b” 等
^匹配字符串的开头^a：匹配以 “a” 开头的字符串
$$`匹配字符串的结尾`b$$：匹配以 “b” 结尾的字符串
*匹配前一个字符 0 次或多次ab*：匹配 “a”、”ab”、”abb” 等
+匹配前一个字符 1 次或多次ab+：匹配 “ab”、”abb” 等
?匹配前一个字符 0 次或 1 次ab?：匹配 “a”、”ab”
``逻辑或操作符`ab`：匹配 “a” 或 “b”
[]匹配括号内的任意一个字符[abc]：匹配 “a”、”b” 或 “c”
[^ ]匹配不在括号内的字符[^abc]：匹配除 “a”、”b”、”c” 以外的字符
{n}匹配前一个字符 n 次a{2}：匹配 “aa”
{n,m}匹配前一个字符 n 到 m 次a{1,3}：匹配 “a”、”aa”、”aaa”
\转义特殊字符\.：匹配 “.” 字符

分组查询

分组查询 是 MySQL 中通过 GROUP BY 子句将查询结果按某一列或多列进行分组，并对每个分组的数据执行聚合函数（如 SUM()、COUNT()、AVG() 等）的查询操作。分组查询通常用于对数据进行统计或分类汇总。

分组查询与 HAVING 子句

  • HAVING 子句 用于过滤分组后的数据。它与 WHERE 不同，WHERE 在分组前过滤行数据，而 HAVING 在分组后过滤分组结果。

示例：

找出玩家金币总和超过 400 的等级：

结果：
| level | total_gold |
|———-|——————|
| 20 | 450.50 |
| 30 | 500.00 |

  • 解释：
    • GROUP BY level：按 level 分组。
    • SUM(gold)：计算每个分组的金币总和。
    • HAVING SUM(gold) > 400：过滤出金币总和超过 400 的分组。

可以按多个列进行分组，实现更精细的分组统计。

| 20 | Charlie | 300.50 |
| 20 | David | 150.00 |
| 30 | Eve | 500.00 |

  • 解释：GROUP BY level, name 会同时根据 level 和 name 进行分组。

分组查询总结

  1. 使用 GROUP BY 子句将数据按指定列分组。
  2. 通常结合聚合函数（如 COUNT()、SUM()、AVG() 等）来计算每个分组的统计结果。
  3. 如果要对分组结果进行过滤，使用 HAVING 子句。
  4. 可以按单列或多列进行分组。

注意事项：

  • 在分组查询中，SELECT 语句中的列必须是分组列或聚合函数的结果。
